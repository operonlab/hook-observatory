Ⅷ 實務混用場景

🚗 自駕車即時系統

Petri Net + Timed Automaton + BT + Actor — 四層混用的實時安全關鍵系統

哲學

自駕是「可證實時性 + 並行處理 + 動作決策 + 隔離故障」四個挑戰同時存在的系統。沒有單一模型能同時解決。

Petri Net 建模感知→預測→規劃→控制的並行管線，可靜態證明無 deadlock。Timed Automaton 為每個步驟加上時間約束，剎車訊號必須在 100ms 內送達，這個性質需要形式驗證工具保證。BT 管駕駛行為選擇，帶優先級的 Selector 讓緊急停車永遠優先。Actor 隔離各模組，某顆 LiDAR actor 崩潰不影響攝影機 actor 繼續運行。

分層架構

Layer 1 (管線層)Petri Net / 感知→預測→規劃→控制並行同步 / 多模態 sensor 融合，靜態可證 deadlock-free
Layer 2 (安全層)Timed Automaton / 實時約束驗證 / 剎車訊號 < 100ms、物體偵測延遲 < 50ms，UPPAAL 形式驗證
Layer 3 (決策層)Behavior Tree / 駕駛行為選擇 / Selector(緊急停車 / 避障 / 變道 / 巡航 / 跟車)
Layer 4 (通訊層)Actor Model / 模組間訊息傳遞 / Sensor Actor / Planner Actor / Controller Actor 各自獨立
資料流Sensor actors → Petri 並行管線 → Timed 驗證實時約束 → BT 選駕駛行為 → Controller actor 輸出
核心優勢pipeline 無 deadlock（Petri 靜態證）+ 實時上限可證（Timed 形式驗證）+ 動作可擴展（BT）+ 故障隔離（Actor）

架構示意

// Layer 1: Petri Net (sensor pipeline)
[Camera] ──▶ (Perception) ──▶ [Fused Perception]
[LiDAR]  ──▶ (Perception) ──▶ (Prediction) ──▶ (Planning) ──▶ (Control)

// Layer 2: Timed Automaton (safety watchdog)
state: NORMAL ──obstacle_detected (within 50ms)──▶ ALERT
state: ALERT  ──brake_signal (within 100ms)──▶ BRAKING
state: ALERT  ──timeout (100ms elapsed)──▶ SAFETY_STOP  // hard constraint

// Layer 3: BT (driving decisions, ticks at 10Hz)
Selector
  ├─ Sequence [collision_imminent] → EmergencyStop
  ├─ Sequence [obstacle_in_path]   → Avoidance
  ├─ Sequence [lane_change_safe]   → LaneChange
  └─ Action → AdaptiveCruise

// Layer 4: Actor (ROS 2 nodes)
SensorActor  ──topic: /perception──▶  PlannerActor
PlannerActor ──topic: /trajectory──▶  ControllerActor

具體例子

例 1：Waymo

多模型融合 + ROS 2 topic / node（Actor 模型）管模組通訊；行為規劃層用 BT 管不同駕駛場景的優先級；安全層獨立 watchdog process 確保實時約束，符合 ASIL-D 要求。

例 2：Baidu Apollo

開源自駕平台，完整採用 Cyber RT（Actor 訊息總線）+ Behavior Tree（planning 模組）+ DAG（Petri-like 管線）架構。代碼庫公開，是學習四層混用最完整的真實參考。

例 3：Tesla Autopilot

FSD v12 前以 BT 管駕駛行為決策；神經網路輸出作為 BT 葉節點的 condition 值；Actor 管感知 / 規劃 / 控制各模組隔離。FSD v12 後把 BT 部分替換為端到端 DL，但 Actor 隔離層保留。

何時採用

系統需要通過 ISO 26262 ASIL-B 以上認證、且有嚴格實時約束（毫秒級）、且有多個可能失效的硬體模組時，四層組合是必要的。若只是機器人原型，Petri + BT 兩層就夠；若是工廠 AGV，去掉 Timed Automaton 也可行。

優缺點

✓ 優點
  • 每層可獨立形式化驗證，滿足 ISO 26262 功能安全認證要求
  • Actor 隔離讓某感測器故障不影響其他模組繼續運行
  • BT 決策行為可由測試工程師獨立測試，不依賴硬體
  • Petri Net 管線設計讓加新 sensor 不影響既有並行邏輯
✗ 缺點
  • 極度工程複雜——四種模型、四套工具鏈同時整合
  • 需要多領域專家（形式方法、即時系統、AI 規劃）
  • Timed Automaton 形式驗證的 state space explosion 問題
  • 四層之間的 latency budget 分配需要細緻的系統設計

代表實作

  • Apollo（百度開源）：Cyber RT Actor + BT + DAG，完整四層，代碼完全公開
  • Autoware（Tier IV）：ROS 2 + BehaviorTree.CPP，開源自駕業界標準平台
  • BehaviorTree.CPP v4：自駕 BT 事實標準，支援 reactive BT 和 subtree hot-reload
  • UPPAAL：Timed Automaton 形式驗證工具，自駕安全層驗證主流學術工具
  • Matlab Stateflow：Timed Automaton 建模 + C 代碼生成，OEM 主流選擇
