Ⅴ 並行 / 通訊

Petri Net Petri Network / Place-Transition Net

Tokens 在 Places 間流動 — 原生並行 + 同步 barrier + 可證 deadlock-free

哲學

Carl Adam Petri 1962 年提出。並行不是特例，而是內建語義：多個 transition 同時 fire，多個 token 同時存在。同步 barrier 自然表達：一個 transition 等到所有輸入 place 都有 token 才能觸發。資源流動是系統的核心語言。

九維屬性

哲學並行資源流動，同步 barrier 天然內建
基本單位Places（圓圈，存 tokens）+ Transitions（條）+ Arcs（有權重）+ Initial marking
狀態位置Marking（所有 place 的 token 分佈）
並行單位多 transition 同時 fire（天然並行）
同步機制Join transition（多輸入 place 全有 token 才觸發）
形式化★★★★☆ 強（reachability / liveness / boundedness 可分析）
適合場景訂單 workflow、製造業 FMS、半導體製程排程、BPMN 底層語義
典型代表CPN Tools、YAWL、Camunda（BPMN）、GreatSPN

具體例子

例 1：電商訂單並行處理
p_order ──▶ t_split ──▶ p_inventory
                    └──▶ p_payment

p_inventory ──▶ t_deduct ──▶ p_inv_done
p_payment   ──▶ t_charge ──▶ p_pay_done

p_inv_done + p_pay_done ──▶ t_ship ──▶ p_shipped

扣庫存和扣款並行執行。t_ship 需要兩個 place 各有一個 token 才能觸發，自動實現同步 barrier。

例 2：生產者-消費者 with 容量限制

用一個 place 表示 buffer 容量（初始 N 個 token）。Producer 每次消耗一個容量 token + 產生一個 item token；Consumer 相反。buffer 滿時 producer 自動阻塞，deadlock-free 可形式化驗證。

優缺點

✓ 優點
  • 並行表達力強，同步 barrier 表達優雅
  • Reachability / liveness / boundedness 可形式化分析
  • BPMN 理論基礎，業界廣泛間接採用
✗ 缺點
  • State explosion：大型系統 marking 空間爆炸
  • 無原生階層（需 Colored / Nested Petri 擴展彌補）
  • 工具偏學術，工程整合難度高

代表實作

  • CPN Tools：Coloured Petri Nets，最強分析工具，支援 data + time
  • YAWL：Yet Another Workflow Language，Petri Net 直接衍生的 workflow 引擎
  • Camunda / Activiti（BPMN）：工業 workflow 引擎，BPMN 語義來自 Petri
  • GreatSPN：隨機 Petri Net，用於性能分析
  • Snoopy：視覺化建模工具，支援多種 Petri Net 變體
