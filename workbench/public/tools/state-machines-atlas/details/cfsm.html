Ⅴ 並行 / 通訊

CFSM Communicating Finite State Machine

多個 FSM 透過 channel 傳訊息 — 最簡單的並行 FSM 延伸

哲學

多個獨立 FSM 各自運行，透過 FIFO channel 交換事件。每個 FSM 保持自己的狀態；channel 是唯一的通訊路徑，狀態不共享。模組化是核心價值：每個 FSM 可以獨立設計、獨立驗證，組合成系統後仍保持可讀性。

九維屬性

哲學多 FSM 模組化組合，channel 是唯一耦合點
基本單位Multiple FSMs + Channels（FIFO queues）+ Send / Receive events
狀態位置每個 FSM 持有獨立當前狀態
並行單位FSM 間非同步並行（各自推進）
同步機制FIFO channel（可選 bounded / unbounded）
形式化★★★★☆ 強（ITU-T SDL 標準、SPIN model checker）
適合場景通訊協議設計、producer-consumer、分散式 handshake
典型代表SDL（ITU-T Z.100）、Promela/SPIN、Go goroutine + channel

具體例子

例 1：TCP 三次握手（簡化）
Client FSM              Channel           Server FSM
CLOSED ──SYN──────────▶ [ch]
                        [ch] ──SYN──▶ LISTEN → SYN_RCVD
SYN_SENT ◀──SYN-ACK──── [ch] ◀──── SYN_RCVD
ESTABLISHED ──ACK──────▶ [ch]
                        [ch] ──ACK──▶ ESTABLISHED

兩個 FSM 各自維護 TCP 狀態，透過 channel 傳遞封包事件。

例 2：Producer-Consumer

Producer FSM 在 READY 狀態產生訊息送入 buffer channel；Consumer FSM 從 channel 取訊息後進入 PROCESSING，完成後回 IDLE。channel bound 決定背壓策略。

優缺點

✓ 優點
  • 建模直觀，可表達大部分通訊協議
  • ITU-T SDL 工業標準，電信業廣泛採用
  • 每個 FSM 可獨立測試後再組合
✗ 缺點
  • 狀態空間 = 所有 FSM 的笛卡兒積（組合爆炸）
  • channel 有無 bound 影響可判定性
  • 大型系統難以完整驗證

代表實作

  • SDL / ITU-T Z.100：電信協議設計標準語言，原生支援 CFSM
  • Promela / SPIN：模型檢查工具，可驗證 CFSM 的 deadlock / liveness
  • Go：goroutine + channel 直接對應 CFSM 概念
  • RoboChart：機器人系統的 CSP / CFSM 建模工具
