Ⅱ 輸出位置

Mealy Machine Mealy Finite State Machine

output = f(state, input) — 輸出綁在 transition 上，反應更快

哲學

輸出是狀態與輸入的函數，在 transition 瞬間產生。同樣的狀態，面對不同輸入，可以給出不同輸出。這讓 Mealy Machine 用更少的狀態表達更豐富的行為，代價是輸出分散在箭頭上，追蹤難度提升。

九維屬性

哲學輸出 = f(狀態, 輸入)，在 transition 瞬間發出
基本單位State + Transition（箭頭標 input / output）
狀態位置單一當前狀態
並行單位無（純序列）
同步機制無
形式化★★★★☆ 強（可與 Moore 互轉，狀態數通常更少）
適合場景序列偵測器、通訊協議握手、串流資料處理、硬體 CRC
典型代表UART 控制器、TCP 握手電路、硬體 stream processor

具體例子

例 1：序列偵測器（"101" pattern）
S0 ──1/0──▶ S1 ──0/0──▶ S2 ──1/1──▶ S0
               （偵測到 "101" 時在最後一個 transition 發出 output=1）

輸出 "1" 發生在 transition 瞬間，比 Moore 版少一個狀態。

例 2：自動販賣機

State 記錄已投入金額，transition 同時帶輸入（按鈕）和輸出（出飲料 / 找零）。同一個「已投 15 元」狀態，按不同按鈕輸出不同飲料。

例 3：CRC 校驗電路

每個輸入 bit 觸發 transition，同時輸出 CRC shift register 的新值。輸出與輸入 bit 直接掛鉤，符合 Mealy 的「即時反應」特性。

優缺點

✓ 優點
  • 狀態數少：相同語義的 Moore 版本通常多一個 state
  • 反應即時：同一個 clock cycle 就輸出，無需等待
  • 適合串流：每個輸入對應一個輸出，一一映射
✗ 缺點
  • debug 難：輸出散落在所有 transition 上，難以一眼掌握
  • 硬體毛刺風險：輸出是組合邏輯，輸入變化可能造成 glitch
  • 狀態圖複雜：每條箭頭都有 input/output 標記，視覺雜亂

代表實作

  • VHDL：Mealy pattern（output 在 transition process 內計算）
  • Verilog：always @(state, input) 組合邏輯 block
  • C（嵌入式）：transition_table[state][input] = {next_state, output}
  • 教學：JFLAP Mealy Machine 模擬器
