Ⅰ Chomsky 階梯

NFA Non-deterministic Finite Automaton

非確定性 FSM — 可同時存在多個 next state，可有 ε 轉移

哲學

允許「多路徑並行探索」。給定 (state, input)，下一個狀態可以是零個、一個或多個。ε 轉移讓機器在不消耗輸入的情況下切換狀態。只要有一條路徑到達接受狀態，整個輸入就被接受。

NFA 和 DFA 識別的語言類別完全相同（皆為正則語言），但 NFA 的狀態數通常遠少於等價 DFA。這讓它成為 regex 編譯器的理想中間表示。

九維屬性

哲學多路徑並行探索，有一條路成功即接受
基本單位δ: Q × (Σ ∪ {ε}) → P(Q)，power set 轉移函數，允許 ε 空轉移
狀態位置同時處於多個狀態的集合（概念上）
並行單位概念並行（多路徑），實作上常轉為 DFA 或模擬
同步機制無（路徑獨立）
形式化★★★★★ 極強（等價於 DFA，等價於正則語言，Kleene 定理）
適合場景regex 中間表示、語法分析前端、模式匹配編譯器
典型代表Thompson NFA、Rust regex crate、Russ Cox re1/re2

具體例子

例 1：正則式 (a|b)*abb 的 Thompson 構造
Thompson 構造法：
  q0 ──ε──► q1 ──a──► q2 ──ε──► q7
  q0 ──ε──► q3 ──b──► q4 ──ε──► q7
  q7 ──ε──► q0   (Kleene star 回環)
  q7 ──ε──► q8 ──a──► q9 ──b──► q10 ──b──► q11 (接受)

每個 regex 運算子（concat、union、star）都有對應的 NFA 片段，Thompson 構造法機械地組合它們。

例 2：ε 轉移讓 regex 分支直觀

對於 regex cat|dog，NFA 從初始狀態用 ε 分叉出兩條路徑，各自匹配 c-a-t 和 d-o-g，最後 ε 合流到接受狀態。用 DFA 寫等價轉移表則需要更多狀態，且無法直接對應 regex 語法結構。

例 3：子集構造法（Subset Construction）
NFA 狀態集合 → DFA 狀態
{q0}      → d0
{q1, q2}  → d1   (ε-closure 後展開)
{q2, q3}  → d2
...

將 NFA 的「當前狀態集合」視為 DFA 的單一狀態。NFA 有 n 個狀態，DFA 最多 2n 個——這是 NFA 的代價。

優缺點

✓ 優點
  • 狀態數通常比等價 DFA 小（指數級壓縮）
  • 直接對應 regex 語法，Thompson 構造法機械化
  • ε 轉移讓模組化組合變得自然
✗ 缺點
  • 直接模擬 NFA 較慢（需追蹤狀態集合）
  • 轉 DFA 時狀態數可能指數爆炸（2n）
  • 與 DFA 識別相同語言類別，無額外表達力

代表實作

  • C：Thompson NFA（原始論文實作，grep 的理論基礎）
  • Rust：regex crate（NFA 模擬 + lazy DFA，線性時間保證）
  • Go：regexp 標準庫（Russ Cox 的 RE2 移植，無 catastrophic backtracking）
  • 教學：Russ Cox 的 re1（200 行 C，完整 NFA 模擬）
