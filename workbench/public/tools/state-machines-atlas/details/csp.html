Ⅴ 並行 / 通訊

CSP Communicating Sequential Processes

Tony Hoare 1978 提出 — Go 的 channel 受它啟發

哲學

並行 = 多個順序 process 透過同步 channel 通訊。Hoare 的名言精確地說明了這個哲學：「Don't communicate by sharing memory; share memory by communicating」。Channel 是進程間唯一合法的通訊路徑，記憶體不共享，race condition 根本不存在。

九維屬性

哲學順序進程 + 同步 channel 通訊，記憶體完全不共享
基本單位Processes + Channels（synchronous by default）+ Events + Choice operators + Parallel composition
狀態位置每個 process 的局部狀態 + channel 中的 pending events
並行單位Parallel composition（P ‖ Q）
同步機制Synchronous rendezvous（送收必須同時就緒）
形式化★★★★★ 極強（traces / failures / divergences 語義，FDR 可自動驗證）
適合場景並行系統設計、協議驗證、Go 程式架構、嵌入式系統
典型代表Go（goroutine + channel）、Occam、FDR4、Clojure core.async

具體例子

例 1：Go 的 CSP 對應
// CSP 的 Process 對應 goroutine
// CSP 的 Channel 對應 Go channel（預設同步）
func producer(ch chan<- int) {
    for i := 0; i < 5; i++ {
        ch <- i   // CSP: c!i  (send on c)
    }
    close(ch)
}

func consumer(ch <-chan int) {
    for v := range ch {
        fmt.Println(v)  // CSP: c?v (receive from c)
    }
}

Go 加入 buffered channel 緩和了 synchronous rendezvous 的嚴格性，但核心模型直接來自 CSP。

例 2：FDR 驗證 deadlock

用 CSP 描述兩個哲學家問題，FDR4 自動搜尋 traces 語義下的 deadlock。找到死鎖後回傳反例路徑，讓工程師修正 choice 順序。

優缺點

✓ 優點
  • 實務最廣用（Go 的設計直接受 CSP 啟發）
  • 理論 + 工程兼備：FDR 可驗證，Go 可執行
  • Deadlock 和 livelock 可用 FDR 自動偵測
✗ 缺點
  • Synchronous channel 有時太嚴（Go 加 buffered 緩和）
  • Traces / failures 語義學習曲線不低
  • FDR 驗證需要學習 CSP 形式語法

代表實作

  • Go：goroutine + channel，CSP 最廣泛的工程實作
  • FDR4：CSP 模型檢查器，可驗證 deadlock / livelock / refinement
  • Occam / Occam-π：直接基於 CSP 設計的語言，用於 transputer 處理器
  • Erlang：async 訊息傳遞（CSP 精神相似但 channel 不具名）
  • Clojure core.async：CSP channel 模型移植到 Clojure / ClojureScript
