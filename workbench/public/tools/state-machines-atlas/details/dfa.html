Ⅰ Chomsky 階梯

DFA Deterministic Finite Automaton

確定性有限自動機 — 每個 (state, input) 只有一個 next state

哲學

無記憶、無分支、最基礎的狀態機。用有限個狀態表達系統，每個 (state, input) 對只有唯一的下一個狀態。理論上，它識別的正是所有正則語言，是 Chomsky 階梯的最底層。

九維屬性

哲學純確定性的狀態轉移，無記憶，無分支
基本單位δ: Q × Σ → Q，有限狀態集 Q，字母表 Σ，初始狀態 q₀，接受狀態集 F
狀態位置單一當前狀態
並行單位無（純序列）
同步機制無
形式化★★★★★ 極強（正則語言，Myhill-Nerode 定理，Brzozowski 極小化）
適合場景詞法分析、正則表達式匹配、協議 parser、UI flow
典型代表flex、RE2、Python re 模組、HTTP header parser

具體例子

例 1：偶數個 1 的 bit string 識別器
       ┌───0───┐       ┌───0───┐
       ▼       │       ▼       │
  [S0]─── 1 ──►[S1]─── 1 ──►[S0]
  (接受)                    (接受)

兩個狀態足以判斷任意長度 bit string 中 1 的奇偶性。S0 為接受狀態（偶數個 1）。

例 2：URL parser

從 SCHEME → HOST → PORT → PATH → QUERY → FRAGMENT，依字元類型推進狀態。每個轉移都是確定的，不需回溯。

例 3：自動門控制
CLOSED ──感應器觸發──► OPEN
OPEN   ──計時到──────► CLOSED

兩個狀態、兩個事件。事件驅動、確定轉移，是 DFA 最日常的工程體現。

優缺點

✓ 優點
  • 最快（常數時間轉移，table lookup 直接實作）
  • 數學性質完整，有 Brzozowski 極小化算法
  • 可靜態分析：可達性、等價性均可機械驗證
✗ 缺點
  • 表達力有限，無法辨識 anbn（需要計數）
  • 從 NFA 轉來時狀態數可能指數爆炸（2n）
  • 無記憶、無堆疊、無遞迴

代表實作

  • C：flex／lex（詞法分析器生成器，從 regex 生成 DFA table）
  • C++：RE2（Google 正則引擎，線性時間保證，拒絕 catastrophic backtracking）
  • Python：re 模組部分實作、automata-lib（教學級 DFA/NFA API）
  • 教學工具：JFLAP（圖形化自動機模擬，適合驗證狀態轉移表）
