Ⅳ 機率 / 時間

Markov Chain Markov Chain / Markov Process

Transition 是機率分布 P(s'|s) — 無記憶性，下一步只看現在

哲學

Markov property：未來狀態只取決於當前狀態，歷史無關。這是最純粹的機率轉移模型，每條邊是一個條件機率，轉移矩陣每行之和為 1。穩態分布存在時，系統長期行為完全可解析。

九維屬性

哲學無記憶性機率轉移 — P(s'|s) 與歷史無關
基本單位State space S + Transition matrix P（行和 = 1）+ 初始分布 π₀
狀態位置單一當前狀態（機率分布形式）
並行單位無（單鏈序列推進）
同步機制無（離散時間步驟驅動）
形式化★★★★★ 極強（穩態分布、mixing time、ergodic theorem 均可解析）
適合場景PageRank、推薦系統、文字生成、天氣建模、MCMC 採樣
典型代表Google PageRank、hmmlearn、PyMC、R markovchain

具體例子

例 1：天氣 Markov 鏈
狀態：{晴, 雨, 陰}
轉移矩陣 P：
       晴    雨    陰
  晴 [0.7,  0.2,  0.1]
  雨 [0.3,  0.5,  0.2]
  陰 [0.4,  0.3,  0.3]

求穩態：π = π · P，解 πP = π, Σπᵢ = 1

三個狀態 + 矩陣乘法，長期天氣比例可解析算出，不需模擬。

例 2：Google PageRank

網頁圖 + 隨機遊走：每個節點是狀態，超連結是轉移邊，加上 0.15 的 teleportation 保持 ergodic。穩態分布即各頁面排名分數。

例 3：字元級文字生成

從語料統計 P(下一字|當前字)，形成轉移矩陣。根據機率採樣下一字，逐步生成文本。Transformer 出現前的主流做法。

優缺點

✓ 優點
  • 數學成熟：穩態分布、mixing time、ergodic theorem 均有閉合解
  • 模擬簡單：每步只需查矩陣 + 採樣
  • MCMC 基礎：Metropolis-Hastings、Gibbs sampling 都建立於此
✗ 缺點
  • 無記憶性是強假設：真實系統常有歷史依賴（語言、股市）
  • 狀態空間可能巨大：連續狀態需要近似（粒子濾波器、變分推斷）
  • 高階依賴難表達：n-gram 模型是 Markov 的延伸但仍受限

代表實作

  • Python：hmmlearn（含 Markov Chain 子集）、PyMC（MCMC 框架）
  • R：markovchain 套件（轉移矩陣 + 穩態解析）
  • 研究工具：Google PageRank 原始實作（稀疏矩陣冪迭代）
  • Julia：MarkovChains.jl（高效能模擬 + 穩態分析）
