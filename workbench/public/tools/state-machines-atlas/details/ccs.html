CCS Calculus of Communicating Systems

Robin Milner 1980 提出 — π-calculus 的前身

哲學

極簡進程代數：只有 send / receive 同步 action + non-determinism，沒有其他原語。Milner 的目標是找到並行系統的最小代數基礎，讓 bisimulation 等價性成為可計算的語義判斷標準。理論純粹到幾乎可以當作並行計算的「λ calculus」。

九維屬性

哲學極簡代數，bisimulation 等價是核心語義工具
基本單位Processes + Actions（a, ā 對）+ Prefix + Choice（+）+ Parallel（|）+ Restriction（\）+ Recursion
狀態位置進程結構（LTS，Labelled Transition System）
並行單位Parallel composition（P | Q）
同步機制Complementary action rendezvous（a 與 ā 配對，產生 τ）
形式化★★★★★ 極強（bisimulation 理論完整，Milner 因此獲 Turing Award）
適合場景協議正確性教學、bisimulation 分析、並行理論研究
典型代表CWB-NC、mCRL2、TAPAs、Milner《Communicating and Mobile Systems》

具體例子

例 1：簡單握手協議
-- Client 送 request，收 response
Client = req.resp̄.Client

-- Server 收 request，送 response
Server = req̄.resp.Server

-- 系統並行組合
System = (Client | Server) \ {req, resp}

-- req 與 req̄ 配對 → τ（內部動作）
-- resp 與 resp̄ 配對 → τ
-- \ 限制兩個 action 只在內部使用

限制算子（\）讓 req / resp 成為內部通訊，外部觀察者看不到；bisimulation 分析確認兩端行為等價。

例 2：Bisimulation 等價性判斷

兩個進程 P = a.(b.0 + c.0)、Q = a.b.0 + a.c.0 在 trace 語義下等價，但 bisimulation 語義下不等價：P 在執行 a 後才能選擇；Q 在 a 之前就已決定分支。CCS 的語義精確區分兩者。

優缺點

✓ 優點
  • 理論最純粹：bisimulation 語義漂亮且完整
  • 是 π-calculus 的前身，理解 CCS 等於理解整個進程代數族
  • 教學效果好：極小語法讓核心概念清晰
✗ 缺點
  • 完全學術，主流工程幾乎不直接使用
  • 工具小眾，工業支援接近於零
  • 無 data types、無 mobility，表達實務系統需要大量擴展

代表實作

  • CWB-NC（Concurrency Workbench）：CCS 互動分析環境，bisimulation 自動判定
  • mCRL2：CCS / μCRL 的後繼工具，工業規模的進程代數分析
  • TAPAs：教學用 CCS 工具，視覺化 LTS 展示
  • Robin Milner《Communicating and Mobile Systems》：CCS + π-calculus 原著
