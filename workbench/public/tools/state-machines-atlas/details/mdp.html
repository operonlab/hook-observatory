MDP Markov Decision Process

Markov Chain + action + reward — 強化學習的理論基礎

哲學

在 Markov Chain 上加入 agent、決策與獎勵信號。MDP 回答一個問題：哪條行動序列能最大化期望累積獎勵？Bellman 最優方程把這個問題轉成遞迴形式，Value Iteration 與 Policy Gradient 都從這裡出發。MDP 是強化學習的理論骨架。

九維屬性

哲學Agent 在機率環境中決策，最大化折扣累積獎勵 E[Σ γᵗ rₜ]
基本單位States S + Actions A + Transition P(s'|s,a) + Reward R(s,a,s') + Discount γ
狀態位置單一當前狀態（agent 完全可觀察）
並行單位無（序列決策，每步一個 action）
同步機制無（Bellman backup 在求解期間迭代傳播價值）
形式化★★★★★ 極強（Bellman 最優方程、Policy Gradient 定理均有嚴格推導）
適合場景遊戲 AI、機器人導航、庫存管理、推薦系統排序、自動駕駛規劃
典型代表OpenAI Gym、Stable-Baselines3、RLlib、DeepMind Acme

具體例子

例 1：Grid World Navigation
狀態：格子座標 (x, y)
動作：{上, 下, 左, 右}
轉移：80% 成功移動，20% 隨機滑動
獎勵：寶藏格 +10，陷阱格 -5，其餘 -0.1
目標：Value Iteration 求最優 policy π*(s)

Bellman 方程迭代收斂後，每格都有最優行動。策略清晰可視覺化，是 RL 教學的經典起點。

例 2：AlphaGo 的 MDP 結構

棋局是 MDP：狀態為棋盤局面，動作為落子位置，轉移確定性（圍棋無隨機），獎勵為終局勝負（+1/-1）。AlphaGo 用 MCTS + Policy/Value Network 近似求解此高維 MDP。

例 3：庫存補貨決策

狀態為當前庫存量，動作為訂貨數量，需求量隨機（transition 有機率性），獎勵為銷售利潤減庫存成本。用 Value Iteration 或 DQN 求出最優補貨策略。

優缺點

✓ 優點
  • RL 核心框架：DQN、PPO、SAC 全都建立在 MDP 形式上
  • Bellman 方程可遞迴求解：小狀態空間有精確解（Value/Policy Iteration）
  • 可結合函數近似：DQN 用神經網路逼近 Q(s,a)，突破 tabular 限制
✗ 缺點
  • 狀態空間大時 tabular 方法失效：需要函數近似（帶來訓練不穩定）
  • Markov property 不總成立：部分場景需要 POMDP 或 Recurrent 策略
  • Reward shaping 有陷阱：設計不當的獎勵函數會導致 agent 鑽漏洞
