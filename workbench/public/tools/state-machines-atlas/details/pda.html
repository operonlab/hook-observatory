Ⅰ Chomsky 階梯

PDA Pushdown Automaton

FSM + 堆疊 — 能辨識 context-free 語言（如平衡括號）

哲學

在 FSM 上加一個無限堆疊當記憶體。堆疊讓機器能「記住」它進入了多少層嵌套——這正是正則語言做不到的事。平衡括號、算術表達式、程式語言語法：這些都是 context-free 語言的範疇，PDA 是它們的識別器。

yacc、bison、ANTLR 的理論根基就在這裡。

九維屬性

哲學FSM + 無限堆疊，記憶嵌套深度，識別 context-free 語言
基本單位δ: Q × (Σ ∪ {ε}) × Γ → P(Q × Γ*)，stack alphabet Γ，堆疊操作 push/pop
狀態位置當前狀態 + 堆疊內容（合稱「格局」）
並行單位無（非確定 PDA 概念上多路徑）
同步機制無（堆疊本身提供隱式同步）
形式化★★★★☆ 強（等價於 CFG，CYK 算法，但 DPDA ≠ NPDA）
適合場景程式語言 parser、表達式求值、XML/JSON 驗證、括號匹配
典型代表yacc/bison（LALR）、ANTLR（LL）、tree-sitter、pyparsing

具體例子

例 1：平衡括號語言 { (ⁿ )ⁿ | n ≥ 0 }
讀到 '('  → push '#' 到堆疊
讀到 ')'  → 若堆疊非空，pop '#'；否則拒絕
輸入結束  → 若堆疊為空，接受；否則拒絕

( ( ( ) ) )
↑ ↑ ↑ ↓ ↓ ↓
push push push pop pop pop → 堆疊空 → 接受

堆疊扮演計數器的角色。純 DFA 無法計數，因此無法識別此語言。

例 2：算術表達式 shift-reduce parser
輸入: 3 + 4 * 2
Shift: 推入 3
Shift: 推入 +
Shift: 推入 4
Shift: 推入 *   (優先級高，繼續 shift)
Shift: 推入 2
Reduce: 4 * 2 → E
Reduce: 3 + E → E

堆疊儲存已讀取的符號片段，reduce 動作將頂端的符號序列折疊為非終端符。yacc 的 LALR 算法本質上就是確定性 PDA。

例 3：Recursive Descent = PDA

手寫的 recursive descent parser，呼叫堆疊（call stack）就是 PDA 的堆疊。每個函數對應一個非終端符，遞迴呼叫對應 push，函數返回對應 pop。Python、C、Go 的標準 parser 都採用這個模式。

優缺點

✓ 優點
  • 足夠表達大部分程式語言語法（CFG 涵蓋範圍廣）
  • yacc/bison/ANTLR 的理論根基，工具鏈成熟
  • 堆疊提供自然的嵌套記憶，遞迴結構直觀對應
✗ 缺點
  • 非確定 PDA（NPDA）表達力強於確定 PDA（DPDA），不對稱
  • 無法表達 anbncn（需要 context-sensitive 語法）
  • LALR 衝突（shift/reduce、reduce/reduce）需手動消除

代表實作

  • C：yacc／bison（LALR(1) parser 生成器，POSIX 標準工具）
  • Java/多語言：ANTLR 4（LL(*) parser，廣泛用於 DSL 設計）
  • C/多語言：tree-sitter（增量 GLR parser，用於編輯器語法高亮）
  • Python：pyparsing、lark（純 Python CFG parser 庫）
