π-calculus Pi Calculus

進程代數 — channel 名稱是一等公民，可動態傳遞

哲學

Robin Milner 在 CCS 基礎上提出 π-calculus：channel 本身可以作為訊息傳遞。這讓系統的通訊拓撲在運行時動態改變 — mobile processes。兩個進程可以交換一個「新 channel 的名稱」，之後透過這個私有 channel 直接溝通，繞開原來的中間人。

具體例子

例 1：Channel 傳遞（mobility 核心）
-- Coordinator 把新的私有 channel c 傳給 Worker
Coordinator = νc . (send(coord, c) | recv(c, result) . done)
Worker      = recv(coord, ch) . compute . send(ch, value)

-- 執行後 Worker 直接透過 c 回報，不再經過 Coordinator 公開 channel

ν（restriction）創造新的私有 channel；傳遞後形成動態點對點連結。

例 2：Spi calculus — Diffie-Hellman 金鑰交換

在 π-calculus 上加入密碼學原語（加密 / 解密），可直接建模和驗證 DH 協議的 secrecy。攻擊者是一個和協議並行的進程，可以竊聽公開 channel。

優缺點

✓ 優點
  • 能建模 mobility：通訊拓撲動態重組
  • Security protocols 的形式化驗證基礎（Spi calculus）
  • 理論優雅：bisimulation 語義統一
✗ 缺點
  • 完全學術，無主流實務語言直接對應
  • 學習曲線極陡：需要紮實的進程代數基礎
  • 工具小眾，工程整合幾乎為零

代表實作

  • Spi calculus：π-calculus + 密碼學原語，安全協議驗證標準工具
  • Mobile Workbench：π-calculus 的互動式分析環境
  • Pict 語言：基於 π-calculus 的實驗性程式語言
  • ProVerif：安全協議自動驗證工具，語義接近 π-calculus
