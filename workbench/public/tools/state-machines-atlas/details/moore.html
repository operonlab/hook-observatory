Ⅱ 輸出位置

Moore Machine Moore Finite State Machine

output = f(state) — 輸出綁在狀態上，進什麼狀態就輸出什麼

哲學

輸出是狀態的函數，與當下輸入無關。進入一個狀態就決定了輸出，不需要再看輸入。這讓每個狀態的語義非常明確：狀態本身就是系統在某個時刻的「描述」，輸出不過是這描述的外化。

九維屬性

哲學輸出 = f(狀態)，與輸入解耦
基本單位State（附帶 output label）+ Transition（不帶 output）
狀態位置單一當前狀態
並行單位無（純序列）
同步機制無
形式化★★★★☆ 強（可與 Mealy 互相轉換，略多狀態）
適合場景數位電路控制邏輯、教學用 FSM、FPGA 狀態控制器、硬體協議序列器
典型代表VHDL/Verilog Moore pattern、FPGA control unit、教科書 FSM 範例

具體例子

例 1：交通號誌燈
State RED    → output: 紅燈亮
State YELLOW → output: 黃燈亮
State GREEN  → output: 綠燈亮

RED ──timer──▶ GREEN ──timer──▶ YELLOW ──timer──▶ RED

燈色由狀態決定，與計時器事件本身無關。每個狀態的輸出一目了然。

例 2：門鈴音符播放器

每個 state 對應一個音符（Ding / Dong / Silence）。按下門鈴觸發 transition，輸出的聲音由當前狀態決定，不看「哪個按鈕」被按。

例 3：數位電路 state machine

VHDL 的 Moore pattern 在每個 clock 邊緣輸出對應 state bits，下游電路直接讀 state register，不需額外解碼輸入組合。

優缺點

✓ 優點
  • 易 debug：輸出綁 state，看狀態就知道輸出是什麼
  • 硬體合成直觀：用 lookup table 直接對映 state→output
  • 沒有組合邏輯毛刺（glitch）風險，輸出穩定
✗ 缺點
  • 狀態數通常比 Mealy 多（需要「記住」要輸出什麼）
  • 反應慢一個 clock cycle：輸出在進入新狀態後才出現
  • 相同行為的 Mealy 版本往往更精簡

代表實作

  • VHDL：Moore pattern（state register + output assignment in process）
  • Verilog：always @(state) 輸出邏輯 block
  • C（嵌入式）：state-indexed output table，中斷驅動
  • 教學：JFLAP（圖形化 Moore/Mealy 模擬）
