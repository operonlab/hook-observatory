Ⅵ 替代模型

Dataflow Dataflow / Reactive

Signal 變化自動傳播 — spreadsheet、React、RxJS 都是這類

哲學

程式 = 資料相依圖。source 變化，下游 derived 自動重算，不需要手動通知。這是 Excel 試算表的本質：你不需要寫「當 A1 改變，重算 B1」，相依關係本身就是程式。核心問題只有一個：追蹤哪些 derived 依賴哪些 source，在 source 變化時最小化重算範圍。

九維屬性

哲學資料相依圖驅動計算，source 變化自動傳播至所有 derived
基本單位Signals / Observables（source）+ Derived（衍生值）+ Effects（副作用）+ Dependency Graph
狀態位置Signal 節點（細粒度，可追蹤每個值的訂閱關係）
並行單位RxJS operator pipeline（異步流）；同步 signal 通常單執行緒
同步機制Push（Observable）vs Pull（Signal lazy eval）；glitch-free 需額外設計
形式化★★★☆☆ FRP 有理論基礎（Conal Elliott 1997），實作語意因框架差異大
適合場景UI 框架、試算表、事件流處理、real-time dashboard、表單驗證
典型代表SolidJS Signals、Vue 3 ref/computed、Svelte 5 runes、RxJS、MobX

具體例子

例 1：SolidJS Fine-Grained Reactivity（Signal 傳播）
import { createSignal, createMemo, createEffect } from "solid-js";

const [price, setPrice] = createSignal(100);
const [qty, setQty] = createSignal(3);

// Derived：自動追蹤 price + qty 的依賴
const total = createMemo(() => price() * qty());

// Effect：total 變化時自動執行
createEffect(() => {
  console.log(`總金額：${total()}`);
  updateCartUI(total());
});

// 只需改 signal，下游自動更新
setPrice(120);  // total 自動變 360，effect 自動執行

SolidJS 的 fine-grained reactivity 不需要 Virtual DOM diff。price 改變，只有依賴 price 的 memo 重算，只有訂閱 total 的 DOM 節點更新。

例 2：RxJS 事件流（Observable pipeline）
import { fromEvent, debounceTime, distinctUntilChanged,
         switchMap, map } from "rxjs";
import { ajax } from "rxjs/ajax";

const search$ = fromEvent(inputEl, "input").pipe(
  map(e => e.target.value),
  debounceTime(300),          // 停止輸入 300ms 才觸發
  distinctUntilChanged(),     // 值沒變不重複搜尋
  switchMap(query =>          // 取消前一次未完成的請求
    ajax.getJSON(`/api/search?q=${query}`)
  )
);

search$.subscribe(results => renderResults(results));

每個 operator 是一個轉換步驟，組合成管線。debounce、取消前次請求、去重，這些複雜的非同步邏輯用宣告式寫法三行搞定。

例 3：Vue 3 Composition API（ref + computed）
import { ref, computed, watchEffect } from "vue";

const firstName = ref("John");
const lastName = ref("Doe");

// computed 自動追蹤依賴，只在相依值變化時重算
const fullName = computed(() => `${firstName.value} ${lastName.value}`);

// watchEffect 等同 createEffect，依賴自動收集
watchEffect(() => {
  document.title = fullName.value;  // title 隨 fullName 自動更新
});

firstName.value = "Jane";  // fullName + document.title 自動更新

Vue 3 的 Proxy-based 響應系統在賦值時攔截，建立依賴關係圖。computed 有快取：若依賴未變，多次讀取只計算一次。

優缺點

✓ 優點
  • 宣告式：寫 what（值的關係），不寫 how（通知流程）
  • 自動最小化重算：依賴圖精準追蹤，只更新受影響的節點
  • UI 思維直覺：試算表、表單這類資料相依場景天然吻合
  • Signals 比 Virtual DOM 效能更好：fine-grained 直接更新 DOM，省去 diff
✗ 缺點
  • debug 困難：相依圖在 runtime 動態建立，傳播路徑難以靜態分析
  • memory leak 風險：Observable subscription 忘記 unsubscribe 會持續持有引用
  • glitch 問題：push 模型中，derived 可能在一個 tick 內收到中間狀態（需 batching 解決）
  • push vs pull 概念差異：Observable（push）與 Signal（pull）混用時語意容易混淆

代表實作

  • JavaScript / Fine-Grained：SolidJS（最純粹的 Signal 實作，無 VDOM）
  • JavaScript / Fine-Grained：Preact Signals（可嵌入 React 生態，3KB）
  • Vue 3：Composition API ref / computed / watchEffect（Proxy-based，全家桶）
  • Svelte 5：Runes（$state / $derived，編譯期 signal，零 runtime overhead）
  • JavaScript / Stream：RxJS（push Observable，事件流管線；Angular 官方採用）
  • JavaScript / State：MobX（OOP 風格 observable，自動追蹤，與 React/Vue 均可整合）
  • 理論起源：Haskell FRP（Conal Elliott 1997，Functional Reactive Programming 學術根源）
