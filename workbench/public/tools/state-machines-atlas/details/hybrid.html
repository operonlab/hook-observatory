Hybrid Automaton Hybrid Automaton / Hybrid System

離散狀態 + 連續動態 — 控制系統的標準建模語言

哲學

物理世界是離散與連續的混合體：汽車有整數擋位（離散），但轉速是連續微分方程。Hybrid Automaton 在每個離散模式（mode）中掛載連續動態（ODE），guard 條件觸發模式切換，reset 重設連續變數。這是控制工程師建模嵌入式控制器的標準方式。

九維屬性

哲學每個離散模式掛載 ODE，guard/reset 觸發模式切換，精確建模物理系統
基本單位Modes（離散狀態）+ Variables（連續）+ ODE per mode + Guards + Resets + Invariants
狀態位置（當前 mode，連續變數向量 x）的組合
並行單位多個 Hybrid Automaton 可並行組合（物理子系統組合）
同步機制共享連續變數 + 離散事件廣播（Stateflow 的 send/receive）
形式化★★★☆☆ 中強（線性 HA 可達性可判定；非線性一般不可判定）
適合場景嵌入式控制器、汽車電控、電力系統、生物神經模型、機器人運動規劃
典型代表Matlab Stateflow、HyTech、SpaceEx、Flow*、dReal

具體例子

例 1：汽車自動變速箱
模式：{Gear1, Gear2, Gear3, Gear4}
連續變數：v（車速）, ω（引擎轉速）

Gear1 ODE：dω/dt = f₁(throttle, v)
Gear2 ODE：dω/dt = f₂(throttle, v)

Guard Gear1→Gear2：ω ≥ 3500 RPM
Reset：ω := ω × gear_ratio

Invariant Gear1：ω ≤ 4000 RPM

每個擋位有不同的動力方程。SpaceEx 可驗證轉速不會超過引擎紅線。

例 2：恆溫器 On/Off 控制

模式 {Heating, Cooling}：Heating 時溫度 ODE 為 dT/dt = α(T_set - T)，Cooling 時為 dT/dt = -β(T - T_env)。Guard 為 T ≥ 25°C 切 Cooling，T ≤ 18°C 切 Heating。HyTech 驗證溫度永遠在安全範圍內。

例 3：彈跳球

模式 {Falling, Bouncing}：Falling 時 dy/dt = v, dv/dt = -g；碰地（y ≤ 0 ∧ v < 0）觸發 reset v := -c·v（彈性係數）。Flow* 可計算最多彈幾次才停止的 reachable set。

優缺點

✓ 優點
  • 精確建模物理系統：離散切換 + 連續動態無縫整合
  • 可做 reachability 分析：線性 HA 有完整工具鏈（HyTech、SpaceEx）
  • Matlab Stateflow 是工業主流：汽車電子、航空電子廣泛採用
✗ 缺點
  • 非線性 HA 不可判定：reachability 一般無法精確求解
  • 數值計算誤差積累：ODE 積分器誤差可能掩蓋真實行為
  • 驗證工具門檻高：SpaceEx / Flow* 設定複雜，非線性支援有限

代表實作

  • MATLAB：Stateflow（工業主流，與 Simulink ODE 引擎整合）
  • C++：SpaceEx（線性 HA reachability，支援大規模系統）
  • C++：Flow*（非線性 HA Taylor 模型 reachability）
  • C / SMT：HyTech（早期線性 HA 驗證器，學術參考）
  • SMT：dReal（δ-decidable 非線性算術，處理混合系統近似驗證）
