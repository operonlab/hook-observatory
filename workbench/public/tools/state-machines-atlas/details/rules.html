Ⅵ 替代模型

Rules Rules Engine

條件匹配觸發 if-then — 無狀態概念，純規則

哲學

規則引擎是聲明式的：你寫「當 X 成立，做 Y」，引擎決定觸發順序。業務邏輯與執行流程解耦。Rete 算法的核心洞見：只對變化的 facts 重新匹配，而非每次掃描所有規則，讓匹配效能從 O(rules × facts) 降至接近 O(changes)。

九維屬性

哲學聲明式條件匹配，業務邏輯與執行流程解耦
基本單位Facts（工作記憶體）+ Rules（if-then）+ Inference Engine（Rete 算法）+ Conflict Resolution

並行單位無（單次推理週期）；部分引擎支援並行規則 firing
同步機制Agenda：衝突規則排隊，按 salience（優先級）或 FIFO 執行
形式化★★★★☆ 有完整推理語意，Rete 算法有理論保證
適合場景保險核保、信用卡反欺詐、電商定價、合規審查、業務規則外包給非工程師
典型代表Drools、CLIPS、Easy Rules、OpenL Tablets、DMN 決策表

具體例子

例 1：保險核保（多條件組合規則）
rule "高風險拒保"
  when
    Applicant( age > 65, smoker == true )
    HealthRecord( chronicDisease == true )
  then
    application.setDecision(REJECT);
    application.setReason("高風險組合");
end

rule "標準費率"
  when
    Applicant( age <= 40, smoker == false )
  then
    application.setPremiumMultiplier(1.0);
end

幾百條核保規則可由精算師直接維護，不需工程師介入。規則間互相獨立，增加新規則不影響現有邏輯。

例 2：電商促銷定價（會員折扣分層）
rule "黃金會員雙十一折扣"
  salience 100  // 優先級最高
  when
    Customer( tier == "GOLD" )
    Order( totalAmount >= 500 )
    Event( name == "1111" )
  then
    order.applyDiscount(0.75);  // 75 折
end

rule "一般會員基本折扣"
  salience 10
  when
    Order( totalAmount >= 300 )
  then
    order.applyDiscount(0.95);
end

salience 控制衝突規則的執行順序。促銷活動上架只需新增規則，下架直接刪除，不動主程式。

例 3：信用卡即時反欺詐（pattern 匹配）
rule "異地連續交易"
  when
    $t1: Transaction( status == APPROVED )
    Transaction(
      cardId == $t1.cardId,
      country != $t1.country,
      this after[0s, 60s] $t1  // CEP 時間窗口
    )
  then
    flagForReview($t1.cardId);
end

結合 CEP（Complex Event Processing）的時間窗口，60 秒內同卡異地交易自動標記。毫秒級回應，適合即時場景。

優缺點

✓ 優點
  • 業務邏輯外化：非工程師可直接維護規則（DMN / 決策表介面）
  • Rete 效能優秀：增量匹配，fact 變化只觸發相關規則
  • 規則獨立：單條規則修改不影響其他規則
  • 可解釋性強：每次決策都有明確的觸發規則可追溯
✗ 缺點
  • 規則衝突難 debug：salience 優先級設計不當會產生意外行為
  • 規則膨脹後難管理：千條以上規則的依賴關係難以全局把握
  • 與 DB 整合有挑戰：working memory 與持久化資料的同步策略複雜
  • 學習曲線：Rete 算法概念、DRL 語法需要時間熟悉

代表實作

  • Java：Drools（Red Hat，最主流；支援 DRL、DMN、決策表）
  • LISP：CLIPS（1986，NASA 使用；規則引擎理論參考實作）
  • Java（輕量）：Easy Rules（嵌入式場景，無 Rete；適合規則數 < 100）
  • 標準規範：DMN 1.4（OMG 決策模型標記法，工具中立）
  • 企業試算表：OpenL Tablets（Excel 定義規則，工程師零介入）
  • 語意網：Apache Jena（OWL / SPARQL 推理，知識圖譜場景）
