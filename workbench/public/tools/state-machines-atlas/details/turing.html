Ⅰ Chomsky 階梯

Turing Machine Turing Machine

FSM + 無限 tape — 理論計算上限，等價於所有可計算模型

哲學

Church-Turing thesis：一切可計算的，都等價於 Turing Machine 能計算的。DFA 加堆疊得到 PDA，PDA 加無限 tape 得到 TM。Tape 可以左右移動、讀寫，這讓它等價於任意隨機存取記憶體。

TM 是理論宇宙的終點，不是工程工具。它的價值在於劃定邊界：停機問題不可判定，意味著某些問題天生無法被任何程式解決。

九維屬性

哲學FSM + 無限讀寫 tape，等價於所有可計算模型，劃定計算邊界

狀態位置當前狀態 + tape 內容 + head 位置（合稱「格局」）
並行單位無（單一確定性版本）；多 tape TM 等價但加速
同步機制無

具體例子

例 1：Unary 加法 Turing Machine
輸入 tape: 111 + 1111  (3 + 4，unary 表示)
目標輸出:  1111111     (7)

步驟：
1. 找到 '+'，將它改為 '1'
2. 移動到 tape 最右端的 '1'，改為 blank
3. 輸出: 1111111 blank

δ(q0, 1)  = (q0, 1, R)   # 跳過左邊的 1
δ(q0, +)  = (q1, 1, R)   # 把 + 換成 1
δ(q1, 1)  = (q1, 1, R)   # 跳過右邊的 1
δ(q1, B)  = (q2, B, L)   # 到達末端
δ(q2, 1)  = (qf, B, R)   # 刪掉最後一個 1

即使是最簡單的加法，Turing Machine 的描述也需要逐格推進。這說明它是理論模型，不是工程工具。

例 2：停機問題（Halting Problem）不可判定
假設存在 Halts(P, I)，能判斷程式 P 對輸入 I 是否停機。

構造程式 D：
  D(x):
    if Halts(x, x): loop forever
    else:           halt

問：D(D) 會停機嗎？
  若停機  → Halts(D,D) = true → D 應該 loop → 矛盾
  若不停機 → Halts(D,D) = false → D 應該 halt → 矛盾

對角線論證。TM 的最重要貢獻之一：不是「能計算什麼」，而是「不能計算什麼」。

例 3：通用圖靈機（Universal Turing Machine）

UTM 接受 (⟨M⟩, w) 作為輸入，其中 ⟨M⟩ 是任意 TM 的編碼，w 是輸入字串。UTM 模擬 M 在 w 上的執行。這是「程式即資料」的理論根源，也是現代計算機（stored-program computer）的數學原型。

優缺點

✓ 優點
  • 能計算任何可計算函數（Church-Turing thesis）
  • 理論完備，是計算複雜度（P/NP）的參照模型
  • 劃定邊界：停機問題確立了「不可計算」的存在
✗ 缺點
  • 實務無用（速度極慢，逐格推進）
  • 停機問題不可判定，無法靜態驗證任意程式終止性
  • 對工程設計幾乎無直接指導意義

代表實作

  • 理論等價：Lambda Calculus（Alonzo Church，與 TM 同年提出，函數式語言根基）
  • 極小語言：Brainfuck（8 個指令，Turing-complete，常用於演示 TM 下界）
  • 細胞自動機：Conway's Game of Life（Turing-complete，但極度低效）
  • 教學模擬器：Turing Machine Simulator（Web 版圖形化 TM 執行，適合驗證轉移函數）
