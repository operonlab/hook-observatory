Ⅰ Chomsky 階梯

DFA Deterministic Finite Automaton

確定性有限自動機 — 每個 (state, input) 只有一個 next state

哲學

最基礎的自動機模型：用有限個狀態表達系統，輸入驅動狀態轉移，輸出由當前狀態決定。無記憶、無分支、無隨機。是正則語言的識別器，也是所有狀態機的理論起點。

九維屬性

哲學純確定性的狀態轉移
基本單位State + Transition (δ: Q × Σ → Q)
狀態位置單一當前狀態
並行單位無（純序列）
同步機制無
形式化★★★★★ 極強（對應 Regular Language，Myhill-Nerode 定理）
適合場景詞法分析、正則表達式匹配、協議解析、簡單 UI flow
典型代表lex/flex、re2、正則引擎、HTTP parser

具體例子

例 1：偶數個 1 的 bit string 識別器
S0 ──1──▶ S1
S0 ◀──1── S1
S0, S1 均接受 0 自轉
S0 = 接受狀態（偶數個 1）

兩個狀態就能判斷任意長度 bit string 的 1 是否為偶數。

例 2：URL parser

從 SCHEME → HOST → PORT → PATH → QUERY 依字元類型推進狀態，每個轉移都是確定的。

優缺點

✓ 優點
  • 最快（常數時間轉移）
  • 可用 table lookup 直接實作
  • 數學性質完整，有極小化算法
✗ 缺點
  • 表達力有限（無法辨識 a^n b^n）
  • 狀態數可能爆炸（尤其從 NFA 轉來）
  • 無記憶、無計數、無遞迴

代表實作

  • C：flex（詞法分析器生成器）
  • C++：RE2（Google 正則引擎）
  • Python：re 模組（部分）、automata-lib
  • 理論工具：JFLAP（教學用自動機視覺化工具）
