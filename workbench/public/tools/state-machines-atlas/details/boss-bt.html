Ⅷ 實務混用場景

🎮 遊戲 Boss 戰三層架構

FSM (階段) → Behavior Tree (決策) → FSM (技能) — 經典 top-down

哲學

Boss 戰有三個不同層次的問題：WHAT（現在在哪個階段）、HOW（這個階段做什麼決策）、WHEN（技能如何執行）。三個問題性質不同，用同一種模型回答都不合適。

外層 FSM 管階段切換——血量 33% / 66% 是硬邊界，清晰、確定。中層 BT 管決策優先級——召喚小怪 vs 範圍技 vs 普攻，帶優先級的選擇邏輯。葉層 FSM 管技能協議——蓄力→施法→冷卻是剛性順序，不允許中途插斷。三層各用最合適的模型。

分層架構

Layer 1 (外層)FSM / 血量觸發階段切換 / Phase1 → Phase2 → Phase3
Layer 2 (中層)Behavior Tree / 階段內 AI 決策 / Selector(補血 / 召喚 / 範圍技 / 普攻)
Layer 3 (葉層)FSM / 技能執行協議 / 蓄力 → 施法 → 冷卻
資料流Boss HP → Phase FSM 觸發切換 → 激活對應 BT → BT 選技能 → Action 節點觸發技能 FSM
核心優勢階段切換清晰（FSM）+ 決策帶優先級（BT）+ 技能剛性流程（FSM），三種需求三種工具

架構示意

// Layer 1: Phase FSM
OnHPChanged(boss):
  if hp < 0.33 * maxHP and phase != Phase3:
    phase = Phase3
    activeBehaviorTree = bt_phase3   // 換 BT

// Layer 2: BT (Phase3 子樹 tick)
Selector
  ├─ Sequence [血量 < 10%] → HealSelf
  ├─ Sequence [cooldown == 0] → SummonMinion
  ├─ Sequence [playerInRange] → ConeAttack
  └─ Action → BasicAttack            // fallback

// Layer 3: 技能 FSM（ConeAttack）
Idle ──trigger──▶ Windup ──done──▶ Cast ──done──▶ Cooldown ──ready──▶ Idle

具體例子

例 1：Dark Souls — 太陽王 Gwyn

三段血量觸發不同 Phase FSM；每個階段內 BT 選擇連擊模式（2 hit、4 hit、grab）；每招的蓄力判定→傷害框→硬直恢復走獨立技能 FSM。設計師在 Phase 2 BT 把 grab 優先級調高，不需改 FSM。

例 2：原神深淵 Boss

機制切換（護盾激活、元素吸收）由外層 FSM 管；BT 在護盾激活期間把「召喚充能」優先级調高；各機制的動作序列走獨立狀態機，確保不被打斷。

例 3：Hollow Knight Boss 系列

每個 Boss 血量閾值對應不同技能組；進入新 Phase 後 BT 根節點換成不同子樹；每個 pattern（衝刺、跳躍、爪擊）走固定狀態序列。開發者在 DLC Boss 加新 Phase 時只需在 FSM 加一個轉移邊。

何時採用

Boss 有明確的血量 / 時間 / 觸發條件切分的「階段」，且每個階段內有 3 種以上可選行為時，這個三層組合最合算。若 Boss 全程只有一個行為模式，一層 FSM 足夠；若階段切換也有條件分支，可考慮把 Layer 1 也換成 BT。

優缺點

✓ 優點
  • 每層專精，職責清晰，單層 bug 不汙染其他層
  • 擴展容易——加新 Phase 只改外層 FSM，加新技能只加 BT 節點
  • Debug 分層，可分別視覺化 Phase 狀態 / BT tick / 技能 FSM
  • 設計師可獨立調整 BT 決策，不用動工程程式碼
✗ 缺點
  • 跨層通訊需要設計（Phase FSM 如何激活正確 BT）
  • 團隊需要同時掌握 FSM + BT 兩種模型知識
  • 簡單 Boss 一層 FSM 足夠，三層是過度工程
  • 工具鏈整合複雜（BT editor + FSM debugger + 技能編輯器）

代表實作

  • Unreal Engine：官方 Behavior Tree 編輯器 + GameplayAbility System（技能 FSM）
  • Unity：NodeCanvas / Behavior Designer（BT）+ Animator State Machine（技能 FSM）
  • Godot：自研 StateMachine node + BehaviourToolkit plugin
  • 開源參考：libgdx-ai（Java BT + FSM），設計與 UE 架構一致
