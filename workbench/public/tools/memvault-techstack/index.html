QueryClassifyOp 判定 intent 後決定 pipeline 深度：
● 必跑：QueryClassifyOp → Fast Search（內含 11-stage Scoring + Reranking）→ fast cards → Output

WRITE TRACK — 記憶存入 event-driven, fire-and-forget
入口閘門 — 三道安檢循序跑

READ TRACK — 記憶召回 可與 Write 完全同時
QueryClassifyOp — 意圖分類
Keyword（~0ms）和 Semantic Vector（~5ms）並行跑，信心分數融合後判定 intent。融合信心仍低時才觸發 LLM（~500ms）。根據 intent 決定 ScoringConfig + RetrievalMode。Slow Thinker 預取結果在此處注入。
並行融合6 intent動態信心閾值prefetch 注入
Fast Search — qdrant_search() — 必跑
Qdrant 混合搜尋（Dense + BM25 RRF）後，在同一函式內部依序跑 11-Stage Scoring Pipeline 和 Attention-Gated Reranking。Scoring 和 Reranking 不是獨立的 pipeline 階段——它們嵌在 qdrant_search() 裡面。結果成為 fast_cards。
必跑Scoring 嵌入Reranking 嵌入→ fast_cards
Cascade Recall — slow intent 才觸發
GLOBAL（L2 Summary + L1 Community）和 LOCAL（L0 Triple + PPR Walk）不走 scoring pipeline。僅 Blocks 透過 qdrant_search() 走完整 scoring + reranking。CRAG Eval 可選。Access Rerank 輕量排序。

所有入口（Hook · API · MCP · CLI）走同一條 pipeline。輸出格式（text · json · cards）是最後一步的序列化參數，跟檢索策略完全無關。

為什麼是兩欄 + 底部？
Write 和 Read 完全獨立——你存記憶的同時可以查記憶。Background Track 永遠在跑。Read Track 的核心：Slow 是 Fast 的超集——Fast Search 永遠跑，Slow 額外加跑 Cascade，兩組 cards 併入同一 response。Scoring 和 Reranking 嵌在 qdrant_search() 內部。KAS 分離後 Block = SSoT——attitude 以 block_type dispatch 併入 fast search。

11-Stage Scoring Pipeline

11 stages
1 Recency Boost
「越新的記憶越重要——像昨天學的比去年學的記得更清楚」
score × (1 + 0.15 × e^(-age/14)) · weight 可由 intent 調整
2 Importance Weight
「信心高的記憶加分——像確定答案的題目分數加倍」
score × (0.7 + 0.3 × confidence) · unset 預設 0.5
3 Trust Boost
「來源可靠的記憶加分——老師教的比路人說的更可信」
score × (1 - 0.3 × (1 - trust)) · MemoryProvenance 來源追蹤
4 Feedback Boost
「你說『對』就加分，說『錯』就扣分——像訓練狗的獎勵」
score × (1 + 0.15 × tanh(net/3)) · 閉環學習
5 Length Normalization
「太長太短都扣分——像作文不能只寫一行也不能寫一百頁」
score ÷ (1 + 0.3 × |log2(len/500)|) · anchor=500 · 雙向懲罰
6 Weibull Decay + Access
「記憶像食物會過期，但常翻閱的衰減更慢——像常走的路不會長草」
4 tier: Core 180d/0.4 · Hot 60d/0.3 · Warm 30d/0.2 · Cold 14d/0.1 · confidence 決定 tier · access_count 延壽
7 PPR Boost
「知識圖譜的中心樞紐加分——像城市的主要幹道比小巷重要」
score × (1 + 0.3 × ppr_score) · HippoRAG 啟發 · entity mention 匹配 · 無 PPR context 時跳過
8 Semantic Boost
「跟你問的問題越像的記憶加分——像找到一模一樣的拼圖碎片」
score × (1 + 0.3 × cosine_sim) · intent-dependent: entity_lookup 0.5
9 Min Score Gate
「分數太低的直接淘汰——像考試不及格的不能進決賽」
threshold 0.10 · hard filter · 低於門檻直接移除
10 Noise Filter
「過濾掉沒營養的對話——像整理筆記時丟掉塗鴉」
text_ops.noise · 7 類: greeting/too_short/heartbeat/status_only/raw_error_trace/raw_json_dump/repetitive · 記憶關鍵字保護
11 Pairwise Dedup
「太像的答案先降分再淘汰——不是直接刪，是漸進式去重」
cosine > 0.85 → score ×0.5 → 再與 min_score 比對淘汰
