# AI-Native 影片製作工作流設計

> 狀態：構想階段（Brainstorming）
> 日期：2026-03-22
> 背景：為多機位拍攝工作室（2-3 人團隊）設計 AI 原生影片製作流水線

## 問題陳述

傳統影片後製是線性瀑布流程：拍攝 → 素材整理 → 粗剪 → 精剪 → 字幕 → 字卡 → 調色 → 輸出。
用 AI 取代其中個別步驟（如字幕生成）只是「用舊思維套 AI」——流程的形狀沒有改變。

核心問題：**如果從零開始，用 AI agent 的思維來設計影片製作流水線，它會長什麼樣子？**

## 三派辯論摘要

透過三個 AI agent 並行辯論，探索了三種架構哲學：

### 語意優先派

- **核心主張**：理解一次，衍生多次。AI 先建立 Semantic Map（結構化 JSON：speaker / topic / emotion / scene_type / importance），所有下游工作從同一張地圖平行展開
- **錨點**：Semantic Map（Single Source of Truth）
- **優勢**：消除重複理解素材的時間；修改成本低（改地圖標記，下游自動重新衍生）
- **弱點**：地圖品質是天花板——STT 錯誤、Speaker ID 誤判會汙染所有下游

### 平行流派

- **核心主張**：素材落地即啟動一切。字幕、場景分析、情緒標記、調色建議、字卡草稿全部平行啟動，人類做最終組裝
- **錨點**：素材儀表板（所有半成品一覽）
- **優勢**：消滅等待空窗；人的決策從流程中間移到最後
- **弱點**：平行化會放大 garbage in garbage out 效應；半成品間缺乏一致性

### 意圖驅動派

- **核心主張**：先定義成品，再讓 AI 反推路徑。人描述 what（風格、節奏、時長、受眾），AI 負責 how
- **錨點**：Intent Brief
- **優勢**：粗剪即第一版成品；修改用自然語言描述差異
- **弱點**：品味無法形式化；Intent Brief → MLT XML 的語意鴻溝是最大技術挑戰

## 收斂結論：三派是三層，不是互斥

```
┌─────────────────────────────────────────────┐
│  Layer 3: Intent（意圖層）                    │
│  "我要一支 8 分鐘節奏快的訪談精華"              │
│  → 給系統方向，但不是每次都需要                  │
└─────────────────────┬───────────────────────┘
                      ▼
┌─────────────────────────────────────────────┐
│  Layer 2: Semantic Map（語意層）               │
│  結構化理解：誰說了什麼、場景、情緒、重要性        │
│  → 所有下游的 Single Source of Truth           │
└─────────────────────┬───────────────────────┘
                      ▼
┌─────────────────────────────────────────────┐
│  Layer 1: Parallel Workers（平行執行層）       │
│  字幕/字卡/調色/粗剪 同時產出                    │
│  → 實際幹活的引擎們                             │
└─────────────────────────────────────────────┘
```

### 分階段上線策略

| 階段 | 上什麼 | 效益 | 難度 |
|------|--------|------|------|
| Phase 1 | 平行執行層 | 即刻省時，素材進來自動跑 STT/場景偵測 | ⭐⭐ |
| Phase 2 | 語意地圖 | 下游從地圖衍生，修改成本降低 | ⭐⭐⭐ |
| Phase 3 | 意圖層 | 寫 Brief 就出片，人變導演 | ⭐⭐⭐⭐ |

## AI 的角色定位：動態角色切換

AI 不是取代人，也不只是工具。是**動態角色切換的協作夥伴**：

```
導演助理 ←──────────────────────────→ 技術顧問
  (建議)         (輔助執行)         (解決問題)

  "這段建議用    "幫我把字幕      "這個轉場效果
   B機位反應"     時碼對上"        MLT做不到，
                                  可以用ffmpeg
                                  預烘焙這樣解"
```

加上第四角色——**學習者**：人手動微調後，AI 記住偏好，下次代入經驗。

## 工具分工：MLT 做骨架，Resolve 做皮膚

### 為什麼混用

| | MLT | DaVinci Resolve Free |
|---|---|---|
| 擅長 | 結構性操作（剪輯、字幕、字卡、時間軸） | 美學性操作（調色、特效、Fusion） |
| AI 控制方式 | XML 直接讀寫（已驗證） | Script API (Python/Lua) |
| Diff 友好度 | ⭐⭐⭐⭐⭐ 純文字 XML | ⭐⭐⭐ API 讀回 |
| 人類 GUI | Shotcut（基本） | Resolve（業界標準） |

### 交接方式

初期採用**渲染交接**（方案 A）：MLT 輸出 ProRes → Resolve 匯入。
理由：最簡單可靠。AI 的剪輯+字幕+字卡做得好的話，人在 Resolve 裡只需專注調色和特效。

### 完整流程

```
素材落地
    │
    ▼
┌──────────── AI Pipeline (全自動) ─────────────┐
│                                                │
│  Whisper STT → 語意分析 → Semantic Map          │
│       │              │           │              │
│       ▼              ▼           ▼              │
│   [字幕生成]    [自動剪輯]   [字卡生成]          │
│       │              │           │              │
│       └──────→ MLT XML 組裝 ←────┘              │
│                      │                          │
│              MLT 渲染 ProRes                     │
└──────────────────────┼──────────────────────────┘
                       │
                       ▼
┌──────────── DaVinci Resolve ─────────────────┐
│                                               │
│  AI 初版調色 (Script API)                      │
│       │                                       │
│  人類精修：調色 + 特效 + 最終審核               │
│       │                                       │

│       │                                       │

```

### AI 學習迴路

```
Round 1:
  AI 生成 timeline (MLT XML + Resolve Script API) → 存 snapshot
  人在 Resolve GUI 微調
  AI 讀回 timeline (Script API) → 新 snapshot
  Diff 兩個 snapshot → 記錄「人改了什麼」
  存入偏好記憶（例：「訪談類開場縮短 30%」「偏好暖色調」）

Round N:
  AI 代入累積偏好 → 生成更貼近導演品味的版本
  人的微調量逐漸減少
```

## 三派共識（確定要建的基礎設施）

1. Ingest 時即啟動分析，不等人
2. Whisper STT + 時碼 = 所有下游的基礎
3. 場景偵測 + Speaker ID = 結構化理解
4. 字幕/字卡/調色平行執行，不串行
5. MLT 動畫限制 → ffmpeg 預烘焙
6. 人 = 審核者/導演，不是操作員

## 已知風險

| 風險 | 嚴重度 | 緩解策略 |
|------|--------|---------|
| Whisper STT 品質（台語/專有名詞） | 高 | 自訂詞彙表 + 人工校對機制 |
| 多機位音訊同步 drift | 中 | 音訊指紋 cross-correlation 對齊 |
| Speaker Diarization 3+ 人精度 | 中 | 重疊語音降級為手動標記 |
| MLT 逐幀動畫限制 | 低 | ffmpeg 預烘焙（已驗證） |
| Resolve Script API 粒度不足 | 待驗 | 需實測 Fusion 控制能力 |
| 「品味」無法形式化 | 持續 | 學習迴路逐步逼近，不試圖完全取代 |
